# `.dlg` 对话脚本 —— 格式参考

[English](DLG_FORMAT.md)

一个商人 = 一个 `BepInEx/config/VisitAPI/<商人id>.dlg` 文件（UTF-8）。**文件名必须等于 24 位十六进制的商人 id。** VisitAPI 启动时自动扫描该目录下的每个 `.dlg`、注册商人、并给它一个突袭外的「对话」按钮（见下）。解析错误会带行号写进 BepInEx 日志（中英双语，跟随游戏语言）。`.dlg` 是运行时资源——改完不用重编，重启游戏即可。

---

## 文件头（在第一个节点之前）

```
trader: <24位十六进制商人id> "显示名"
start: <重复进入的根节点>                          # 默认: root
first: <首次见面节点>                              # 可选
when: level>=4 -> <节点>                          # 按等级/好感度选根节点（见下）
when: level>=3, standing>=0.05 -> <节点>          # 多个条件用逗号连（须全部满足）
trigger: raid <地图> (x, y, z) [door 宽x高[x转角]] [dist 3] [radius 1.2] [once|repeat] "提示文字"
trigger: hideout <区域> (x, y, z) [node <节点>] [dist 3] [if 任务=状态[/状态]] [free] "提示文字"
quest <别名> = <24位十六进制任务id>               # 任务别名；之后凡是要任务 id 的地方都能写 <别名>
```

- `trader:` 必须是商人 id（与文件名一致）；引号里的名字可选、仅显示用。
- **`start:`** 是普通的重复进入根节点（默认节点名 `root`）。**`first:`** 是只播一次的首次见面节点——在突袭/藏身处里只播一次（按商人+档案记录）；**突袭外的「对话」按钮永远跳过 `first:`、直接进根节点系列。**
- **`when:`** 决定玩家落到哪个根节点，条件 `level>=N` / `level<=N` / `standing>=N` / `standing<=N`（多个用逗号连，须全满足）。**按文件顺序第一个命中的生效**；都不命中则用 `start:`。`level`=玩家等级，`standing`=你与该商人的好感度。「对话」按钮**和**突袭/藏身处入口都走这套。
- **`quest <别名> = <id>`** 定义短别名；之后凡是要任务 id 的地方（`accept:`、`if:`、触发器的 `if` 等）都能写别名。可写在任意头部行。

### 触发器

`trigger:` 行在世界里放一个「拜访」交互点。只有**类型**和**地图/区域**是必填的，其余顺序随意。

**`trigger: raid <地图> (x,y,z) …`**
- `<地图>` 与本局地图 id 做（不区分大小写的）子串匹配。常见 id：`factory4_day` / `factory4_night`、`bigmap`(海关)、`Woods`(森林)、`Shoreline`(海岸)、`Interchange`(立交桥)、`RezervBase`(储备站)、`Lighthouse`(灯塔)、`TarkovStreets`(街区)、`Sandbox` / `Sandbox_high`(新手地带)、`laboratory`(实验室)。
- `(x,y,z)` = 必填的世界坐标。站到点位上按 **F11**——BepInEx 日志会打出可直接粘贴的值。
- `door 宽x高[x转角]` = 门式碰撞盒（宽×高×可选 Y 转角）。`dist N` = 最大距离（默认 3）。`radius N` = 视线锥命中半径（默认 1.2）。`once`（默认，首次拜访后消失）/ `repeat`（每局都出现）。
- **多行 `trigger: raid` = 多个点位**（不同地图，或同图多处），各自独立记录。

**`trigger: hideout <区域> (x,y,z) …`**（当处于藏身处时生成）
- `(x,y,z)` = 绝对坐标（F11 取）。本版本里 `<区域>` 仅作信息（定位靠坐标）。
- `node <节点>` = 直接在该节点打开对话（如从任务线中段续起）。`dist N` = 最大距离。`if 任务=状态[/状态]` = **仅**当任务处于这些状态时显示（否则隐藏）。
- `free`（或 `door`）= 自由站立的「看向即交互」：用于没有原生区域菜单的开放点位（需要你**看向**该点）。不加 `free` 时，拜访动作会**合并进**原生区域菜单（如情报中心菜单）。
- **同坐标、不同 `if` 门控的两行 `trigger: hideout`** = 「在此接取下一个任务 / 回此处交差」模式（门控互斥，同一时刻只显示一个）。

## 节点

```
<节点名> bg: 背景文件
> 旁白行 [| bg: 背景文件]                          # 字幕框逐条播放、点击推进；行尾可选 | bg: 该句单独换背景
NPC 台词行
- 选项文本 -> 目标
- 选项文本 -> 目标 | 指令, 指令...
-> 目标                                            # 节点级自动跳转（用于纯旁白节点）
```

- **节点名**只允许字母 / 数字 / `_` / `.` / `-`（这样行内的 `[动作]` / `<富文本>` 不会被误判成节点头）。
- **`bg:`** 纯文件名在 `config/VisitAPI/backgrounds/` 下查找；含 `/` 或 `\` 的值当作相对 `config/VisitAPI/` 的路径。省略 `bg:` 则沿用上一个背景。支持：**图片** `.png .jpg .jpeg` 和**循环视频** `.mp4 .webm .m4v .mov`（默认静音）。
- **`>` 旁白**在原生字幕框逐条播放、点击推进；放完再渲染 NPC 台词 + 选项。只有 `>` 行加一条 `-> 目标` 的节点，旁白结束后自动跳到该节点。
- **逐句背景**：在某条 `>` 行末尾加 `| bg: 文件`（如 `> 她推开门走了进来。 | bg: Standoff.png`），播到这句时切到该背景；没写 `| bg:` 的句子沿用上一张背景。节点头的 `bg:` 是进入节点时的初始背景，逐句 `| bg:` 在其基础上随旁白切换。文件名解析规则与 `bg:` 相同。
- **NPC 台词行** = 任意普通行。一个节点有多条普通行时，**最后一条**作为 NPC 文本显示（多行铺垫请用 `>` 旁白）。
- **`- 选项文本 -> 目标`** = 可点选项。**目标**：节点名 / `@start`（根节点）/ `@trade` / `@tasks` / `@services`（打开突袭外交易界面的对应标签）/ `@close`（或 `@leave`）。**省略 `-> 目标` 则在执行完该选项指令后关闭对话。**
- 文本支持 `{player}` / `{playerName}`（替换成角色昵称）。
- `#` 或 `//` 开头的行是注释；空行随意。

> `@trade` / `@tasks` / `@services` 仅**突袭外**有效（从「对话」按钮打开的对话叠在交易界面之上）。突袭内不跳转（选项停在原节点）。

## 选项指令（在 ` | ` 之后，用 `,` 或 `|` 分隔）

| 指令 | 作用 | 仅在此状态自动显示 |
|---|---|---|
| `accept: <任务>` | 接取任务 | AvailableForStart |
| `handover: <任务> [标签]` | 打开原生物品上交窗口 | Started |
| `complete: <任务>` | 完成任务 + 发奖励 | AvailableForFinish |
| `setstatus: <任务>[=状态]` | 不接取/不完成、直接设状态（裸写 = AvailableForFinish） | — |
| `if: <任务>=状态[/状态]` | 显式显示条件（覆盖上面的自动显示） | — |
| `ifnot: <任务>=状态[/状态]` | 任务处于这些状态时隐藏 | — |
| `standing: <增量>` / `standing: <商人id>=<增量>` | 增减商人好感度，如 `+0.03` / `-0.08`——**不经过任务系统** | — |
| `once` | 选过一次后永不再显示该选项 | — |
| `always` | 取消自动显示门控（用于隐藏任务流程） | — |

- **自动显示门控**：`accept:` / `handover:` / `complete:` 选项仅当任务处于对应状态时显示，除非加 `always`。若一个节点的**所有**选项都被门控掉，则改为**全部显示**（死锁保护——0 选项节点会卡死玩家，因为对话不能用 ESC 关闭）。
- **`complete:`** 在任务还欠 `handover` 物品时会被**拒绝**（防止部分上交就强制完成）；此时选项停在原节点。
- **组合动作**：一个选项只能有一个主动作（`accept` / `handover` / `complete`）。主动作之后可再加 `accept:` 作*修饰符*，同时接取另一个任务，如 `| complete: usb, accept: mats, always`。
- **`standing:`** 是修饰符、不是主动作——可与同选项的 `accept:` / `complete:` 共存。默认作用于本对话商人；`standing: <24位十六进制商人id>=+0.01` 指向别的商人（写原始商人 id，**不是**任务别名）。好感度不做 clamp；正向增量建议配 `once` 防刷。
- `<任务>` 可以是头部别名（`quest xx = …`）或完整 24 位 id。**状态**名：`Locked / AvailableForStart / Started / AvailableForFinish / Success / Fail`，或数字 `0`–`5`。

## 突袭外「对话」按钮

每个有 `<id>.dlg` 的商人，会在其突袭外交易界面自动获得一个**对话**按钮（锚定顶部居中，可在 `TalkButton.*` 配置）。点击在 `when:` 选出的根节点打开对话（跳过 `first:`）。之后 `@trade` / `@tasks` / `@services` 切换交易界面的标签。

---

## 任务状态值

| 数值 | 名称 |
|---|---|
| 0 | Locked |
| 1 | AvailableForStart |
| 2 | Started |
| 3 | AvailableForFinish |
| 4 | Success |
| 5 | Fail |

## 备注

- **取坐标**：突袭/藏身处内按 **F11**，把 `Camera.main` 位置打到 BepInEx 日志，可直接粘进 `trigger:` 行。
- **保留键**：`tab:` 和 `random:` 能解析不报错，但本版本**未启用**（旧版遗留）。不要依赖它们。
- **分隔符**：` -> ` 和 ` | `（两侧带空格）会被当作分隔符——对话文本里要避开。
- **语言**：VisitAPI 自己生成的文本（`(结束)` / `继续…` 行、对话按钮默认文字、默认拜访提示、解析错误）自动跟随游戏语言。`.dlg` 内容本身由你自己写——想双语就两边都写，单语也行。
